## Installation Methods

<Tabs>
  <Tab title="Via Proxmox Web UI">
    1. Navigate to your storage in Proxmox web interface
    2. Click **CT Templates**
    3. Click **Templates** button
    4. Select desired Turnkey template from list
    5. Click **Download**
    6. Create container using downloaded template

    **Note**: This downloads official Turnkey LXC templates from Proxmox template repository.
  </Tab>

  <Tab title="Via Command Line">
    Download template directly:
    ```bash
    pveam update
    pveam available --section turnkeylinux
    pveam download local debian-XX-turnkey-appname_VERSION_amd64.tar.gz
    ```

    Create container:
    ```bash
    pct create 100 local:vztmpl/debian-XX-turnkey-appname_VERSION_amd64.tar.gz \
      --hostname turnkey-app \
      --memory 2048 \
      --net0 name=eth0,bridge=vmbr0,ip=dhcp \
      --storage local-lvm
    ```
  </Tab>

  <Tab title="Via Helper Script">
    Use the interactive Turnkey selector script:
    ```bash
    bash -c "$(wget -qLO - https://github.com/community-scripts/ProxmoxVE/raw/main/turnkey/turnkey.sh)"
    ```

    This script provides:
    - Interactive menu to select from available Turnkey templates
    - Automatic template download from official repository
    - Guided container creation with sensible defaults
    - Password generation and configuration
  </Tab>
</Tabs>

## Comparison: Turnkey vs Helper Scripts

<Tabs>
  <Tab title="Turnkey Templates">
    **Pros:**
    - Official, well-tested configurations
    - Comprehensive management tools included
    - Backed by Turnkey Linux organization
    - Regular security updates
    - Extensive documentation

    **Cons:**
    - May include extra packages you don't need
    - Slightly larger footprint
    - Update cycle may lag latest app versions
    - Limited customization during install
  </Tab>

  <Tab title="Helper Scripts">
    **Pros:**
    - Minimal installation
    - Latest application versions
    - Highly customizable
    - Optimized for specific use cases
    - Interactive configuration

    **Cons:**
    - Less standardized
    - May require more manual configuration
    - Community-maintained (varying support levels)
    - Need to manage updates yourself
  </Tab>
</Tabs>

## Backup & Migration

Turnkey's TKLBAM tool provides:

```bash
# Initialize backup profile
tklbam-init

# Create backup
tklbam-backup

# Restore backup
tklbam-restore

# List backups
tklbam-list
```

<Accordion title="Backup Best Practices">
  - Enable automatic daily backups
  - Store backups offsite (cloud or remote server)
  - Test restore procedures regularly
  - Document custom configurations
  - Exclude temporary files and caches
</Accordion>

## Common Tasks

<AccordionGroup>
  <Accordion title="Change Passwords">
    ```bash
    # Root password
    passwd

    # MySQL root password
    mysqladmin -u root -p password newpassword

    # PostgreSQL password
    sudo -u postgres psql
    \password postgres
    ```
  </Accordion>

  <Accordion title="Update System">
    ```bash
    # Update package lists
    apt update

    # Upgrade all packages
    apt upgrade

    # Full distribution upgrade
    apt dist-upgrade
    ```
  </Accordion>

  <Accordion title="Configure SSL">
    Via Webmin:
    1. Navigate to Webmin → Webmin Configuration → SSL Encryption
    2. Upload certificate or use Let's Encrypt
    3. Apply and restart services

    Via Command Line:
    ```bash
    # Install Let's Encrypt
    apt install certbot

    # Obtain certificate
    certbot certonly --standalone -d your-domain.com
    ```
  </Accordion>

  <Accordion title="Access Logs">
    ```bash
    # System logs
    journalctl -f

    # Apache logs
    tail -f /var/log/apache2/error.log

    # Application logs (varies by app)
    tail -f /var/log/appname/error.log
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't Access Web Interface">
    1. Verify container is running: `pct status VMID`
    2. Check IP address: `pct exec VMID -- ip addr`
    3. Verify firewall allows connection
    4. Check service status: `pct exec VMID -- systemctl status apache2`
    5. Review logs for errors
  </Accordion>

  <Accordion title="First Boot Configuration Didn't Run">
    Manually trigger initialization:
    ```bash
    pct enter VMID
    /usr/lib/inithooks/firstboot.d/run
    ```
  </Accordion>

  <Accordion title="Forgot Admin Password">
    Reset via console:
    ```bash
    # Enter container
    pct enter VMID

    # Reset root password
    passwd

    # Reset application password (varies by app)
    # WordPress example:
    mysql wordpress -e "UPDATE wp_users SET user_pass=MD5('newpass') WHERE ID=1;"
    ```
  </Accordion>
</AccordionGroup>
